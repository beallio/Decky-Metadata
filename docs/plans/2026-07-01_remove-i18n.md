# Plan: Remove the i18n layer and inline English strings (remove-i18n)

## Context

The plugin targets an English-only device and the maintainer wants the i18n layer removed
entirely (not just the Italian block). `src/i18n.ts` exports a single `t(key)` helper backed by
a `STRINGS = { en: {...}, it: {...} }` table; it is imported in **4 files** (`src/index.tsx`,
`src/contextMenuPatch.tsx`, `src/components.tsx`, `src/steam.ts`) with **~195** call sites and
**127** English keys.

Almost all call sites are **literal** `t("KEY")` and must be replaced with the corresponding
English string from `STRINGS.en`. There are exactly **3 dynamic-key** sites in
`src/components.tsx` that build the key at runtime and need small inline English maps instead:
- ~line 864: `t(\`achievementCache_${policy}\` as any)` — English values:
  `hourly:"Hourly", daily:"Daily", weekly:"Weekly", pc_session:"PC session", manual:"Manually"`.
- ~line 1500: `t(\`achievementSource_${source}\` as any)` — English values:
  `auto:"Auto", retroachievements:"RetroAchievements", xbox:"Xbox", disabled:"Disabled"`.
- ~line 1170: `t(retroResolutionMessageKey(payload?.reason) as any)` — `retroResolutionMessageKey`
  (in `components.tsx`) maps a reason to a `retroDetect*` key; convert it to return the English
  **string** directly (sourced from `STRINGS.en`'s `retroDetect*` values).

**Intended outcome:** `src/i18n.ts` is deleted, no `t(...)` / i18n import remains anywhere, and
every previously-localized string is the same English text inlined at its call site. No
user-visible change (English was already the effective language on the device). The build
(`tsc --noEmit` + rollup) is the safety net — after removal, any missed `t(` reference is a hard
compile error.

**Relevant files:** `src/i18n.ts` (delete), `src/index.tsx`, `src/contextMenuPatch.tsx`,
`src/components.tsx`, `src/steam.ts` (inline strings + drop the `import { t } from "./i18n"`).

**Out of scope:** the delisted refresh UI (its own plan, authored to use literal strings after
this lands); any behavior/logic change; `main.py`; renaming the strings' wording.

**Slug used throughout this plan:** `remove-i18n`

## Implementation Tasks

Frontend-only. Because this is ~195 replacements, do the bulk change with a **deterministic
script** (not hand edits) so every string exactly matches `STRINGS.en`; then handle the 3
dynamic sites, imports, and deletion by hand. No behavior change. No npm deps.

1. **Snapshot the English map.** Before deleting anything, programmatically read the `en` object
   from `src/i18n.ts` into a `{ key: "English text" }` map (e.g. a Node script that imports/`require`s
   or regex-extracts the `en` block; the values may be multi-line concatenations — evaluate them
   so escapes/quotes are preserved exactly).

2. **Codemod the literal call sites.** For each of the 4 files, replace every occurrence of
   `t("KEY")` (and `t('KEY')`) with the JSON-encoded English string `map[KEY]` (i.e.
   `JSON.stringify(map[KEY])`), preserving surrounding context. This is valid in both JS and JSX
   expression positions: `{t("done")}` → `{"Done"}`, `body: t("saved")` → `body: "Metadata saved"`.
   Do **not** touch the 3 dynamic sites in this pass (their argument is a template/among a call,
   not a bare string literal, so a `t("literal")` matcher won't hit them).

3. **Convert the 3 dynamic sites** in `src/components.tsx`:
   - `achievementCache_${policy}`: add a module-scope
     `const ACHIEVEMENT_CACHE_LABELS: Record<string, string> = { hourly: "Hourly", daily: "Daily",
     weekly: "Weekly", pc_session: "PC session", manual: "Manually" };` and replace the call with
     `ACHIEVEMENT_CACHE_LABELS[policy] ?? policy`.
   - `achievementSource_${source}`: add
     `const ACHIEVEMENT_SOURCE_LABELS: Record<string, string> = { auto: "Auto",
     retroachievements: "RetroAchievements", xbox: "Xbox", disabled: "Disabled" };` and replace with
     `ACHIEVEMENT_SOURCE_LABELS[source] ?? source`.
   - `retroResolutionMessageKey(reason)`: change each `return "retroDetectX";` to
     `return "<English text from STRINGS.en.retroDetectX>";` (copy the exact English values), and
     change the call site `t(retroResolutionMessageKey(payload?.reason) as any)` to
     `retroResolutionMessageKey(payload?.reason)`.

4. **Remove the import** `import { t } from "./i18n";` from all 4 files (index.tsx:6,
   contextMenuPatch.tsx:43, components.tsx:48, steam.ts:24). If any file used `currentLang`/other
   i18n exports — it does not (only `t` is exported/imported) — nothing else to remove.

5. **Delete `src/i18n.ts`.**

6. **Verify no residue:** `grep -rnE "\bt\(\"|from \"\./i18n\"|from \"\.\./i18n\"|i18n" src/` returns
   nothing (except perhaps unrelated substrings — ensure no `t(` string-key calls and no i18n
   imports/paths remain). `tsc --noEmit` and the rollup build must pass; a leftover `t(` would be
   a compile error since `t` no longer exists.

7. **Scope discipline:** only string inlining + the 3 dynamic conversions + import removal + file
   deletion. Do not reword any string, change component logic, matching, patches, or `main.py`.
   The bulk-edit script itself may be left in the repo only if placed under a scratch/ignored
   path; otherwise remove it after running (do not commit a stray codemod script).

8. Record a session log under `docs/agent_conversations/` per `AGENTS.md` §9, listing the file
   count and that the change is a mechanical English-inlining with no behavior change.
